Авторизация

Сессия — httpOnly cookie _sid (JWT), выставляется при signup/login. Токен отдельно нигде не возвращается в теле ответа.

POST/api/auth/signup{ email, password } → { id, email }
POST/api/auth/login{ email, password } → { id, email }
POST/api/auth/logout→ {}
POST/api/auth/request-reset-password{ email } → письмо со ссылкой
POST/api/auth/reset-password{ id, token, password } → { status }

Asana

Базовый словарь поз. Список читается либо из in-memory кэша, либо напрямую из БД, если кэш ещё не прогрет — на фронте разницы нет.

AsanaСущность
idnumber
namestringНазвание на русском.
aliasstringТехнический slug, camelCase, напр. adhomukhaShvanasana.
descriptionstring | null
searchKeysstring | nullСинонимы/ключевые слова для поиска, одной строкой через запятую.
imagestring | nullПолный CDN-URL картинки.
groupsAsanasGroup[]includeПрисутствует только там, где явно подгружается связями (см. ниже).
Служебные записи-разделители

В словаре асан намеренно есть две нефункциональные записи для использования в билдере последовательностей: alias: "separator" («Разделитель») и alias: "empty" («Пустой блок»). Это обычные строки Asana, никакого отдельного типа/флага у них нет — определяются по алиасу.

GET/api/asanas/getAll→ Asana[] (с groups)
GET/api/asanas/:id→ Asana
POST/api/asanas/createmultipart/form-data + image файл
PUT/api/asanas/:idmultipart/form-data
DELETE/api/asanas/:id

Sequence

Последовательность из асан. Порядок блоков и асан внутри блока — порядком элементов массива, как и в модели пранаямы.

SequenceСущность
idnumber
titlestring | null
descriptionstring | null
userIdnumberВладелец.
blocksAsana[][]read onlyТолько в ответе GET /:id — см. форму ниже. При создании/обновлении блоки передаются в другом, «плоском» виде.
// GET /api/asanas-sequences/:id — блоки сгруппированы, каждая асана несёт свои флаги
{
  "id": 12,
  "title": "Утренний комплекс",
  "description": "",
  "userId": 2,
  "blocks": [
    [
      { "id": 4, "name": "Собака мордой вниз", "alias": "adhomukhaShvanasana",
        "inRepeatingBlock": false, "inDynamicBlock": true }
    ],
    [
      { "id": 12, "name": "Воин 1", "alias": "vira1",
        "inRepeatingBlock": true, "inDynamicBlock": false }
    ]
  ]
}
// POST /api/asanas-sequences/create и PUT /api/asanas-sequences/:id — тело запроса
{
  "title": "Утренний комплекс",
  "description": "",
  "blocks": [
    [{ "asanaId": 4, "inRepeatingBlock": false, "inDynamicBlock": true }],
    [{ "asanaId": 12, "inRepeatingBlock": true, "inDynamicBlock": false }]
  ]
}

PranayamaSequence

Полностью серверная версия сущности из вашего артефакта с описанием клиентской модели — та же форма данных, теперь с id/userId/таймстампами от бэкенда и явным position у шагов вместо порядка в массиве.

PranayamaSequenceСущность
idnumber
namestring
descriptionstring | null
userIdnumberВладелец.
createdAtstringISO-дата.
updatedAtstringISO-дата.
stepsPranayamaStep[]Только в ответах GET /:id, create, update — не в списке /my.
PranayamaStepВложенная сущность
idnumber
positionnumberПорядок этапа, с 0. Задаётся сервером по порядку массива steps при сохранении.
namestring
descriptionstring | null
mode"timer" | "metronome"enum
durationMode"time" | "cycles"enum
timerSecondsnumber
cycleCountnumber
rhythmstring | nullnull при durationMode: "time". Иначе 2–4 сегмента через :, напр. "4:4:4:4"; сервер отклонит (500) другое количество сегментов или нечисловые значения.
// POST /api/pranayama-sequences/create — тело запроса (id/position/userId сервер проставит сам)
{
  "name": "Утренняя практика",
  "description": "Бодрящая последовательность",
  "steps": [
    { "name": "Капалабхати", "mode": "timer", "durationMode": "time",
      "timerSeconds": 180, "cycleCount": 12, "rhythm": null },
    { "name": "Самавритти", "mode": "metronome", "durationMode": "cycles",
      "timerSeconds": 300, "cycleCount": 12, "rhythm": "4:4:4:4" }
  ]
}
PUT заменяет шаги целиком

Как и у Sequence, обновление пересобирает весь список steps заново — частичного патча одного шага нет. Присылайте полный массив на каждое сохранение.

GET/api/pranayama-sequences/my→ PranayamaSequence[] (без steps)
GET/api/pranayama-sequences/:id→ со steps
POST/api/pranayama-sequences/create
PUT/api/pranayama-sequences/:idполная замена шагов
DELETE/api/pranayama-sequences/:id
